# Round 006 critique: painterly recovery is the decision, not a finishing step

All three proposals converge on path 3. The remaining decision is whether path 3 can preserve the spike's illustrated quality without destroying the geometric and temporal consistency for which 3D was introduced. A pilot that merely produces a cottage and walk sheet does not answer that question. It must compare raw deterministic rendering and any stylized result, rerun them from recorded inputs, measure cleanup time, and judge them in the shipping renderer at fixed scale.

## Claude proposal

### Steelman

The strongest version of Claude's proposal is that a shared 3D scene should own the invariants that image generation handles poorly, while a structure-conditioned 2D model should contribute only surface character. If the repaint can preserve silhouettes, anchors, lighting, and frame identity across a whole atlas while reproducibly matching the spike, this is stronger than my deterministic-only recommendation because it directly targets the largest remaining fidelity gap instead of hoping NPR materials can imitate a composed illustration.

### Attack

The proposal calls cross-asset consistency "guaranteed by the shared 3D source," but the guarantee ends at the raw render. A generative repaint is a new generation boundary that can alter roof pitch, window count, timber placement, hems, hands, foot contact, cast-shadow direction, and apparent volume. Conditioning constrains a sample; it does not make the output an identity-preserving transform. Repainting a tiled sheet may reduce independent sampling, but it creates other failure modes: details can bleed across cells, the model can treat the sheet as one composition, and corresponding features can still mutate between frames. Applying the pass to textures instead of frames is materially different and much safer, but it no longer supports the claim that 2D repaint recovers the final spike look. Those options cannot be grouped as interchangeable mitigations.

The determinism analysis is too narrow for the dispatched requirement. Even if the constitution's strict position rule does not directly govern art pixels, this pilot must be reproducible. A hosted image model identified only as "the image generator" does not provide a stable model version, sampler, seed semantics, service implementation, or byte-stable output. Recording a prompt and seed is provenance, not reproducibility. A generative repaint can be admitted only if its exact accepted outputs are frozen and the pipeline clearly distinguishes archival reproducibility from regeneration reproducibility. For animated production, the stronger requirement is rerunnable temporal behavior. A service call that may yield a different walk sheet cannot be a required build step.

The fallback weakens the recommendation rather than protecting it. If conditioning is unavailable, raw 3D plus hand or generated texturing is not evidence that the painterly half is solved. It is the deterministic NPR path under another name, and it needs its own acceptance result. The pilot therefore needs at least two explicit candidates from the same scene: a deterministic render/composite and a repainted version. Otherwise a failed repaint can be rhetorically converted into a successful path 3 pilot even though the proposal's load-bearing fidelity mechanism failed.

The Meshy economics are acknowledged but not measured. "Minimal retopo/decimate" assumes the fixed camera hides topology defects, while the player must survive eight facings and deformation, where bad edge flow, self-intersections, texture seams, and skin weights are exposed. Buildings also need clean silhouettes from gameplay occlusion angles, even if rendered at one canonical yaw. One cottage and one humanoid are biased samples: the cottage tests hard-surface silhouette, while the player tests rigging, but neither tests foliage, fences, carts, modular variants, or the large inn that dominate a 200-asset cleanup curve. The pilot should log human minutes by category, rejected generations, manual mesh edits, texture replacement, rig repair, and render retries. Without that ledger, "beats column C" says nothing about whether production is economical.

The scale contract is directionally right but incomplete. Meters in Blender do not become pixels by themselves. The contract also needs orthographic camera scale, output resolution, pixels per meter on the ground plane and vertical axis, asset canvas policy, footprint dimensions, contact-plane definition, and a prohibition on downstream resampling. A manifest field named `pixels_per_meter` plus a capture assertion can still pass two assets that were rendered with different camera scale or later normalized to different canvas heights. Door, eave, and person ratios should be validated from scene geometry before render and from known projected landmarks after processing. The proposal also asserts that `atan(0.5)` is the exact angle `projection.gd` assumes without demonstrating the coordinate convention and vertical projection mapping. That needs a primitive calibration scene against the current projection, not an angle copied into prose.

Blender headless is a reasonable choice, but the proposal treats it as settled without accounting for version pinning, color management, render engine, GPU versus CPU behavior, font or texture dependencies, and alpha/shadow compositing. A committed `.blend` file alone is not a render specification. The pilot needs a pinned Blender version and an explicit command that can render on CI or a documented production host. This is still preferable to a Godot sub-viewport for authoring and pass export, but the resulting sprites must be validated in Godot because color management and filtering can change the final read.

The acceptance montage is not sufficiently controlled. The spike is a composed scene while the proposed pilot has one cottage and player, so differences in crop, background, ground dressing, exposure, and asset density can dominate a subjective vibe vote. Scott judging an unblinded A/B/C also invites source and novelty bias. The gate should use matched composition, shipping zoom, fixed color management, and both static and moving captures. Reviewers should score anonymized candidates against named criteria. It must separately fail on painterly resemblance, frame stability, geometric preservation, scale, and cleanup time. "Beat the current build" is too low a bar because the current build contains known flat ground and proxy animation defects. The target is the spike, and passing against a deliberately weak baseline does not prove that target survives decomposition.

Finally, diagnosing `Instance base is null` from guessed `load()` sites is premature. The proposal correctly says it is independent, but a guarded fallback could hide a broken script base or resource parse failure. The fix should start from the engine stack and clean-import reproduction, then add a regression assertion against the actual cause. It should not be bundled into evidence for the art path.

## Antigravity proposal

### Steelman

The strongest version of Antigravity's proposal is a deliberately small vertical slice: use Meshy to test source economics, automate one locked offline renderer, apply only low-strength stylization if raw output misses the target, and feed ordinary sprites into the existing pipeline. Its restraint about stylization is better than making generative repaint mandatory, provided "low strength" is replaced with measurable preservation and temporal criteria.

### Attack

The proposal avoids the central decision by making stylization optional and saying the pilot will reveal "how heavy" it must be. Strength is a model-specific control, not a quality guarantee. Even a low-strength img2img pass can move high-contrast foot edges, facial features, roof contours, and shadows between frames. More importantly, there is no threshold at which the pass is declared too destructive or too nondeterministic. The proposal needs to choose among deterministic NPR/compositing, texture-space generation, sheet-level repaint, and per-frame repaint, then define a rejection rule. Leaving all of them available ensures some attractive frame can be shown without proving a production method.

Its strongest factual overclaim is that a skeletal rig "mathematically cannot drift its anchor or mix up its legs" and that facings will "perfectly align." A rig can have root motion, an offset origin, asymmetric poses, foot sliding, bad inverse kinematics, retargeting errors, skinning collapse, uneven frame sampling, and inconsistent crop normalization. Correct bone identity prevents the semantic left leg from becoming the right leg in the source skeleton, but an orthographic projection can occlude them and a stylizer can repaint them. Anchor stability comes from root-motion policy, contact events, render origin, canvas policy, and postprocessing tests, not from the existence of bones.

The offline renderer choice is unresolved in a way that blocks estimation. Blender headless and a Godot sub-viewport are not equivalent implementation details. Blender offers mature import, rigging, baking, render passes, and color control; Godot offers direct agreement with runtime import and shading but would add editor/render tooling near the game and provides a weaker asset-cleanup workflow. Supporting both doubles calibration and creates two possible sources of truth. The pilot should choose Blender for authoring and batch pass export, then use Godot only for acceptance capture. Any proposal for the reverse needs evidence that Godot can import the chosen Meshy formats, animations, materials, and alpha/shadow passes consistently in headless mode.

The scale contract is only a slogan. "A meter is a meter" does not specify whether Meshy imports preserve units, whether transforms are applied, how a 2 by 2 footprint relates to the authored grid, where the contact plane lies, how the orthographic camera maps meters to pixels, how tall sprites are padded, or whether the current processor rescales them. No numerical player, door, eave, ridge, footprint, pixels-per-meter, or tolerance values are supplied. This omission makes the side-by-side incapable of distinguishing a correct scale pipeline from two assets manually adjusted until they look plausible.

Meshy is treated as a four-hour API task rather than an escalation-class production dependency. The estimate omits account provisioning, cost caps, license and provenance review, API version drift, download retention, source hashing, failed generations, retopology, UV repair, material replacement, armature cleanup, weight painting, animation licensing if Mixamo is used, and the local Blender dependency. Mixamo is itself another external service and content-license question, not a parenthetical implementation option. No second dependency should enter the pilot by convenience. A standard gait should be authored or drawn from already approved, locally retained inputs unless Scott explicitly approves otherwise.

The 16-hour estimate is therefore not credible. Six hours for a robust renderer may cover a happy-path script, but not import variance, pinned tool setup, camera calibration, transparent edges, shadow passes, deterministic color output, atlas assembly, and tests. Five hours for pilot iteration cannot include Meshy rejection cycles plus cottage cleanup, humanoid rig repair, animation, stylization experiments, Godot integration, and review. The estimate also has no scale-out model. At 200 assets, even one hour of manual cleanup per accepted asset is five worker-weeks before variants, animation, review, and rerenders. The pilot must produce observed per-category throughput and an extrapolation range.

The acceptance gate is too vague and too easy to pass. A side-by-side that "confirms the scale is unified and the painterly vibe survives" does not define matched crop, textured ground, shipping zoom, motion review, eight-facing uniqueness, anchor tolerance, temporal boiling, lighting agreement, or who judges it. It also does not compare raw and stylized outputs, so it cannot tell whether 3D or img2img produced the improvement. The pilot should include a fixed in-engine dressed plot, an anonymized static comparison, a walk GIF, a frame-difference or landmark report, and a full rerender from committed local inputs. It should fail if the deterministic render misses the spike and stylization cannot close the gap without visible mutation.

The null-instance diagnosis is again speculative. Calling it a dangling runtime UI or process reference without a stack is not evidence, and a one-hour allocation assumes the cause. It should remain an independent fast lane with clean-import reproduction and an engine-error regression test.

## Required synthesis constraints

Path 3 remains the best pilot, but neither peer proposal is ready as written. Phase 3 should adopt these constraints:

1. Use Blender headless as the single offline authoring and render source, pinned with explicit camera, color, pass, alpha, and CPU render settings. Use Godot only for shipping-renderer acceptance.
2. Establish the scale contract with scene geometry and projected landmark checks before any Meshy asset is judged. Prove camera agreement first with primitives.
3. Produce deterministic NPR/composited output as the baseline. Treat any generative stylization as a separate experimental candidate, never as an implicit finishing step.
4. If stylization is tested, freeze exact accepted outputs and provenance, state honestly whether regeneration is reproducible, and reject any method that causes landmark mutation or temporal boiling. Per-frame independent repaint should be out of scope.
5. Record cleanup labor, rejection count, service cost, and manual interventions separately for the cottage and player. Extrapolate the observed range to buildings, characters, foliage, and props before authorizing production.
6. Run a matched, anonymized in-engine gate against the spike, not merely against the known-bad current build. Score painterly fidelity, structural preservation, motion stability, scale, grounding, and production economics as separate pass conditions.
7. Keep `src/sim/` untouched and keep the null-instance repair independent from the art-path verdict.
