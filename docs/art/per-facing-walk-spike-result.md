# Per-facing walk spike result

The revision cap was three produced generations per facing. The spike stopped
after using all three revisions on the first facing, side.

Identity and costume coherence passed human review. All three outputs read as
the same traveler: the hair, face, build, green tunic, cream sleeves, brown
trousers, backpack, palette, proportions, and rendering style remained
consistent with the reference and with one another.

Locomotion failed human review before the unchanged gate could run on an
assembled 3 by 4 sheet:

- Revision 1 produced alternating silhouettes but painted both boots the same
  color in contact frames, breaking anatomical marker identity.
- Revision 2 improved the top row but still painted both boots the same color
  in the bottom row.
- Revision 3 preserved exactly one magenta boot and one cyan boot in every
  frame, but repeated the cyan-leading contact arrangement in all four poses.
  It did not produce opposite anatomical boots leading.

The staged checkpoint required a usable side strip before generating down and
up. No provisional 3 by 4 source sheet was assembled, so the unchanged gate
was not run and there is no in-game 160 px capture to claim. Reordering or
recoloring these frames could manufacture a chromatic result without proving
anatomical alternation, so neither was used.

Option B therefore failed on alternation at its first facing. Option C remains
the authorized next path. The previously generated side row from revision 3
of the original spike is still the only generated source row that passed the
unchanged gate.
